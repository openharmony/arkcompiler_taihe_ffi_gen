# Taihe 版本演进规格

## 1. 全局规则

- 支持新增全局函数和自定义类型。
- 不支持删除已有全局函数和自定义类型。

## 2. function

- 不支持修改已有全局函数签名（包括名称、参数类型、返回类型、参数顺序等）。
- 可以更改已有全局函数的实现。

## 3. struct/union

- 不支持新增、删除或修改字段/变体（包括名称、类型、顺序等），原因：
  1. Taihe struct/union 是按值传递的，修改字段/变体会影响内存布局，导致二进制不兼容。
  2. 当增加了新字段/变体的 struct/union 被传递给旧版本代码时，若要拷贝/移动/析构该 struct/union，由于旧版本代码无法识别新增的字段/变体，可能导致内存泄漏或未定义行为。

## 4. enum

- 不支持修改枚举值的类型（如不能从 `i32` 修改为 `i64`）。
- 不支持修改原有枚举项（包括名称、值、顺序等）。
- 支持新增枚举项，新增的枚举项必须放在所有旧枚举项的后面，原因：
  1. 枚举类型在 ABI 中被表示为一个整数索引值，索引值对应枚举项在枚举定义中的顺序，修改顺序会导致索引值不兼容。
- 考虑到未来可能新增枚举常量，为保证安全，在文档说明中需要提示用户必须明确处理未知枚举常量的情况。

## 5. interface

- 不支持新增、删除或修改接口继承的父接口。
- 支持在已有接口中新增方法，且新增方法位置无限制。但是，新增方法必须提供默认实现，此外还需要标记接口的当前版本号和方法的最低支持版本号。详细版本控制规则见下文。
- 支持更改已有方法的默认实现。
- 不支持删除和修改已有方法的签名或顺序。

### 5.1 interface 版本控制规则

通过 `@version("VERSION")` 来标记接口当前版本号，其中 `VERSION` 由 1~4 个使用点号分隔的 16 位无符号整数组成。如 `1`、`2.0`、`65535.65535.65535.65535` 等，比较版本号时，应逐段比较各整数值，缺省的整数值视为 0。如果未标记版本号，则视为初始版本 `0.0.0.0`。该版本号会在对象实现接口时被记录在相应的函数虚表中，表示该对象所实现的此接口的实际版本号。

通过 `@min_version("VERSION")` 来标记方法的最低支持版本号，`VERSION` 的格式同上。如果未标记 `@min_version`，则视为该方法从接口的初始版本开始支持。

当调用新增方法时。调用方会先检查被调用对象所实现的相应接口实际版本号是否**不低于**方法的最低支持版本号，若是则调用对象所实现的该方法，否则调用该方法的默认实现。具体细节见 [Taihe 接口方法调用规则](./InterfaceAbi.md#233-方法调用)。

接口的版本演进示例：

```rust
// 未标记版本号，视为初始版本
interface IFoo {
  // 默认表示当前方法从初始版本开始支持
  func0(): void;
} 

// 标记当前 interface 为 1.0 版本
@version("1.0")
interface IFoo {
  func0(): void;

  // 标记当前方法从 IFoo 1.0 版本开始支持
  @min_version("1.0")
  func1(): void;
}

// 标记当前 interface 为 2.0 版本 
@version("2.0")
interface IFoo {
  func0(): void;

  @min_version("1.0")
  func1(): void;

  // 标记当前方法从 IFoo 2.0 版本开始支持
  @min_version("2.0")
  func2(): void;
}
```
