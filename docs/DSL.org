#+TITLE: 接口合约描述语言

* 包

** 定义包
1. 源文件和包一一对应
   - 不支持拆分一个包为多个源文件
   - 也不支持在同一个源文件内描述多个包

2. 包名由源文件名唯一确定
   - 例： =ohos.hardware.sensors.taihe= 源文件内，描述了 =ohos.hardware.sensors= 包下的各个成员

3. 包的成员名称不应和包自身的名称相同
   - 例：若在 =ohos.hardware.sensors= 下定义了 =class SensorManager= ，则不允许定义 =ohos.hardware.sensors.SensorManager.taihe=
   - 理由：JS、Python 等动态类型语言不支持重名

** 使用包管理命名空间
1. 默认可使用当前包下的名称
   #+BEGIN_SRC rust
     // example.types.taihe
     struct Foo {}
     struct Bar {}
     fn func1(bar: example1.Foo): (); // OK
     fn func2(foo: Foo): ();          // OK
   #+END_SRC

2. 可使用 =use= ... =as= 语句导入其他包并生成别名，或使用 =from= ... =use= 语句为其他包下定义的名称生成别名
   #+BEGIN_SRC rust
     // example.test1.taihe
     use example.types as myfoo;
     fn func3(foo: myfoo.Foo);        // OK

     // example.test2.taihe
     from example.types use Bar, Foo as Foobar;
     fn func5(foo: Bar): ();          // OK
     fn func4(foo: Foobar): ();       // OK
   #+END_SRC

3. 包与包之间互相隔离，无从属关系
   #+BEGIN_SRC rust
     // example.taihe
     from types use Foo;                    // Error: package `types` not exist
     use types as mytypes;                  // Error: package `types` not exist
     from example.types use Foo;            // OK
     use example.types;                     // OK
     fn func6(foo: Foo): ();                // OK
     fn func7(foo: example.types.Foo): ();  // OK
   #+END_SRC

* 基础类型
内部提供了以下几种基础类型：
- 无符号整型：=i8=, =i16=, =i32=, =i64=
- 有符号整型：=u8=, =u16=, =u32=, =u64=
- 浮点型：=f32=, =f64=
- 布尔型：=bool=
- 字符串类型：=String=

* 函数
支持在 class/interface 以外定义函数

** 函数参数
1. 函数可以有任意数量的参数，每个参数由参数名、参数类型和类型修饰符共同组成。

2. =mut= 关键是是唯一的类型修饰符，其用于表示该参数是可变的。该关键字不可用于枚举类型和基础类型。

** 返回值
1. 函数可以有零个、一个或多个返回值。每个返回值由返回值的类型表示。

以下是一些合法或非法的函数接口声明示例：
#+BEGIN_SRC rust
  struct Foo {
    a: i32;
    b: i32;
  }
  enum Bar {
    X = 1;
    Y = 2;
  }
  fn func(m: i32, n: i32): (String, i32);  // OK
  fn func(foo: mut Foo);                   // Error: return value type cannot be omitted
  fn func(foo: mut Foo): ();               // OK
  fn func(bar: mut Bar): String;           // Error: `mut` qualifier cannot be used on enum type
#+END_SRC

* 结构体

结构体是若干数据成员的组合，其内部可包含的数据成员类型包括基础类型、枚举类型和其他结构体类型，其接口声明方法如下：
#+BEGIN_SRC rust
  struct Foo {
    a: i32;
    b: i32;
  }
#+END_SRC

* 其他

函数和类型声明没有先后顺序，但需要注意结构体之间不可循环引用。例如：
#+BEGIN_SRC rust
  struct Foo {
    bar: Bar;
  }
  struct Bar {
    val: i32;
  } // OK

  struct Foo {
    bar: Bar;
  }
  struct Bar {
    foo: Foo;
  } // circular reference detected
#+END_SRC
