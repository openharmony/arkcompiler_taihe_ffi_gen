#+TITLE: 接口合约描述语言

* 包

** 定义包
1. 源文件和包一一对应
   - 不支持拆分一个包为多个源文件
   - 也不支持在同一个源文件内描述多个包

2. 包名由源文件名唯一确定
   - 例： =ohos.hardware.sensors.taihe= 源文件内，描述了 =ohos.hardware.sensors= 包下的各个成员

3. 包的成员名称不应和包自身的名称相同
   - 例：若在 =ohos.hardware.sensors= 下定义了 =class SensorManager= ，则不允许定义 =ohos.hardware.sensors.SensorManager.taihe=
   - 理由：JS、Python 等动态类型语言不支持重名

** 使用包管理命名空间
1. 默认可使用当前包下的名称
   #+BEGIN_SRC rust
     // example.types.taihe
     struct Foo {}
     struct Bar {}
     fn func1(bar: example1.Foo): ();       // OK
     fn func2(foo: Foo): ();                // OK
   #+END_SRC

2. 可使用 =use= ... =as= 语句为其他包生成别名，或使用 =from= ... =use= 语句为其他包下定义的名称生成别名
   #+BEGIN_SRC rust
     // example.test1.taihe
     use example.types as myfoo;
     fn func3(foo: myfoo.Foo);              // OK
     
     // example.test2.taihe
     from example.types use Foo as Foobar, Bar;
     fn func4(foo: Foobar): ();             // OK
     fn func5(foo: Bar): ();                // OK
   #+END_SRC

3. 包与包之间互相隔离，无从属关系
   #+BEGIN_SRC rust
     // example.taihe
     from types use Foo;                    // Error: package `types` not exist
     from example.types use Foo;            // OK
     use types as mytypes;                  // Error: package `types` not exist
     use example.types as mytypes;          // OK
     fn func6(foo: types.Foo): ();          // Error: package `types` not exist
     fn func7(foo: example.types.Foo): ();  // OK
   #+END_SRC
